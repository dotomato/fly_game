# 情侣飞行棋 💕

基于 Web 的实时多人棋盘游戏。2–4 名玩家通过分享房间链接加入，轮流掷骰子在 80 格赛道上前进，每个格子都有对应的任务要完成。

## 功能特性

- **实时联机** — 基于 Socket.io，2–4 人同房，状态实时同步
- **分享链接加入** — 自动生成房间链接，一键复制分享；扫链接自动填写房间号
- **多剧本支持** — 创建房间时选择剧本（普通 / NTR / 极限），任务从 `data/scripts/` 读取
- **自定义棋子 emoji** — 加入前从 40+ 个 emoji 中选择自己的棋子
- **80 格任务棋盘** — 每格任务可自由定制，第 75–80 格为终点格
- **流畅动画** — 棋子在格子间飞行移动，弹簧缓动效果
- **实时聊天** — 气泡样式文字聊天，支持保存为截图
- **语音消息** — 长按 🎤 录音（最长 60 秒），点击播放
- **断线重连** — 游戏中或等待室断线后 5 分钟内可自动重连，恢复状态
- **聊天记录存档** — 每局游戏结束后，完整聊天记录（含系统消息）保存至服务器
- **响应式布局** — 支持桌面端和移动端

## 快速开始

```bash
npm install
npm start
# 访问 http://localhost:3000
```

## 部署

```bash
bash deploy.sh
```

推送代码到 GitHub，然后 SSH 连接生产服务器执行拉取、安装依赖、重启服务。

## 项目结构

```
fly_game/
├── server.js          # Express + Socket.io 后端（所有游戏逻辑）
├── deploy.sh          # 一键部署脚本
├── data/
│   ├── scripts/       # 剧本任务配置（JSON）
│   │   ├── couples.json
│   │   ├── couples_ntr.json
│   │   └── ntr_extreme.json
│   └── chat_logs/     # 每局游戏聊天记录（自动生成）
└── public/
    ├── index.html     # 大厅（创建 / 加入房间 + 等待室）
    ├── game.html      # 游戏棋盘
    ├── game.js        # 前端逻辑
    └── style.css      # 样式
```

所有游戏逻辑（掷骰子、位置计算、轮次切换、终点判断）均在服务端执行，客户端只负责发送事件和渲染状态。

## 游戏规则

1. 创建或通过分享链接加入房间（2–4 人）
2. 房主点击**开始游戏**（房主始终第一个投骰子）
3. 轮到自己时，点击 🎲 掷骰子（1–6 点）并前进
4. 落子后完成该格显示的任务，点击**确认完成**
5. 第 75–80 格为终点格，落入并确认后淘汰
6. 所有人淘汰后游戏结束

## 剧本配置

剧本文件位于 `data/scripts/`，每个剧本是一个 JSON 文件：

```json
[
  { "id": 1, "content": "任务描述", "hasEnd": false },
  { "id": 80, "content": "终点任务", "hasEnd": true }
]
```

第 75–80 格必须设置 `"hasEnd": true`。修改后需重启服务器生效。

新增剧本时，在 `server.js` 顶部的 `SCRIPTS` 数组中注册：

```js
const SCRIPTS = [
  { id: 'couples', name: '普通版', file: 'couples.json' },
  // 在此添加新剧本
];
```

## 技术栈

- **后端**：Node.js、Express、Socket.io
- **前端**：原生 JS、CSS 自定义属性、Socket.io 客户端
- **语音**：Web MediaRecorder API（格式优先级：webm/opus → webm → wav）
- **截图**：html2canvas
